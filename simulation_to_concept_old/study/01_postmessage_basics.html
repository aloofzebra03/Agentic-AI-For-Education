What is PostMessage?

The postMessage() API enables cross-origin communication between:

  • A page and an iframe embedded in it
  • A page and a popup window it opened
  • Two iframes in the same page

Basic Syntax

Sending a Message:

// Send to parent window (from iframe)
window.parent.postMessage(data, targetOrigin);

// Send to iframe (from parent)
iframe.contentWindow.postMessage(data, targetOrigin);

// data: Any JSON-serializable object
// targetOrigin: '*' (any) or specific URL like 'https://example.com'

Receiving a Message:

window.addEventListener('message', function(event) {
    // SECURITY: Always verify the origin!
    if (event.origin !== 'https://trusted-site.com') {
        return; // Ignore messages from unknown origins
    }
    
    console.log('Received:', event.data);
    console.log('From origin:', event.origin);
});

For Your Simulation

To make your simulation notify the parent Streamlit app:

// Add this to your simulation's JavaScript

// Function to notify parent of parameter changes
function notifyParamChange(paramName, oldValue, newValue) {
    const message = {
        type: 'SIMULATION_PARAM_CHANGE',
        param: paramName,
        oldValue: oldValue,
        newValue: newValue,
        allParams: getCurrentParams(), // Your function to get all params
        timestamp: Date.now(),
        source: 'pendulum_simulation'
    };
    
    // Send to parent window
    window.parent.postMessage(message, '*');
    
    console.log('Sent to parent:', message);
}

// Example: Call when length slider changes
document.getElementById('lengthSlider').addEventListener('input', function(e) {
    notifyParamChange('length', previousLength, e.target.value);
    previousLength = e.target.value;
});

Security Best Practices

  1. Always validate origin - Check event.origin before processing
  2. Validate data structure - Ensure received data has expected format
  3. Use specific targetOrigin - Avoid '*' in production
  4. Don't trust data blindly - Sanitize before using
